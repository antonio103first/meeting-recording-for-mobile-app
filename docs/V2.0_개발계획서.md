## 1. 개요

### 1.1 목적

V1.x에서 구축한 녹음 → STT → 요약 → 저장 파이프라인을 확장하여, 멀티 AI 엔진 지원, 유연한 요약 방식 선택, STT 변환파일 기반 재요약 기능을 추가한다. 사용자가 상황에 맞는 최적의 엔진과 요약 포맷을 자유롭게 조합할 수 있는 플랫폼으로 업그레이드한다.

### 1.2 현재 상태 (V1.x)

| 구분 | 현재 지원 | 비고 |
|------|-----------|------|
| STT 엔진 | CLOVA Speech, Gemini | 2종 |
| 요약 엔진 | Gemini, Claude | 2종 |
| 요약 방식 | 화자중심, 주제중심, 공식양식(MD), 공식양식(텍스트), 강의요약 | 5종 |
| API 설정 | 단일 스크롤 화면에 Gemini/CLOVA/Claude 나열 | 탭 미사용 |
| 파이프라인 | MP3 → STT → 요약 → 저장 (일괄 처리만 가능) | 단일 흐름 |
| 저장경로 | 앱 전용 디렉토리 고정 | 변경 불가 |
| Drive 업로드 | 녹음파일/회의록 2개 폴더 | STT 별도 폴더 없음 |

### 1.3 V2.0 목표

| 구분 | V2.0 목표 | 변경사항 |
|------|-----------|----------|
| STT 엔진 | CLOVA, Gemini, ChatGPT, Claude | +2종 추가 |
| 요약 엔진 | CLOVA, Gemini, ChatGPT, Claude | +2종 추가 |
| 요약 방식 | 화자중심, 주제중심, 회의양식, 강의요약, 흐름중심 | 이름변경+1종 추가, 1종 삭제 |
| API 설정 | 탭 기반 UI (Clova/Gemini/ChatGPT/Claude) | UX 개선 |
| 파이프라인 | MP3→STT→요약 + STT파일→재요약 (2가지 흐름) | 신규 기능 |
| 저장경로 | 사용자 지정 가능 (폴더 선택 다이얼로그) | 신규 기능 |
| Drive 업로드 | 녹음파일 / STT변환·회의록요약 — 2개 폴더 지정 | 폴더 지정 개선 |

### 2.2 녹음/변환 메뉴 기능 확장

**변경 파일**: `RecordingScreen.kt`, `RecordingViewModel.kt`

#### 2.2.1 현재 파이프라인 (유지)

```
[MP3 녹음/선택] → [변환 시작] → STT 변환 → 요약(설정 기본값) → 저장 → Drive 업로드
```

이 기본 흐름은 그대로 유지한다. 설정에서 정한 STT 엔진, 요약 엔진, 요약 방식이 기본 적용된다.

#### 2.2.2 신규: STT 변환파일 기반 재요약

```
[회의록 메뉴]
  ├─ STT 변환파일 선택 (파일 브라우저)
  └─ 변환 시작
       └─ 요약방식 선택 다이얼로그
            └─ 요약 실행 → 파일이름 입력 → 저장
```

**상세 동작**:

1. **STT 변환파일 선택**: 저장 디렉토리 내 `.txt` 파일 목록 표시 또는 파일 선택기
2. **변환 시작 버튼**: 탭 시 요약방식 선택 다이얼로그(BottomSheet) 표시
3. **요약방식 선택**: 5종 중 택 1 (화자중심/주제중심/회의양식/강의요약/흐름중심)
   - 다이얼로그 열릴 때 **현재 설정에서 선택된 요약방식이 기본 선택(하이라이트)** 상태로 표시
   - 사용자가 다른 방식을 선택하면 해당 방식으로 전환 (설정값 변경 없이 1회성 적용)
4. **요약 실행**: 선택한 방식으로 요약 엔진 호출 (설정의 기본 요약 엔진 사용)
5. **파일이름 입력**: 기본값 `2026-0322_투자심사_흐름중심.md` 포맷
6. **저장**: 요약방식별 개별 파일 저장 (같은 STT에서 여러 방식으로 요약 가능)

**파일 이름 규칙**:
```
{날짜}_{원본파일명}_{요약방식}.md
예: 2026-0322_투자심사_흐름중심.md
    2026-0322_투자심사_회의양식.md
    2026-0322_투자심사_화자중심.md
```

**UI 구성**:
- 기존 "회의록 요약 결과" 카드 하단에 구분선 추가
- "STT 변환파일로 재요약" 섹션 추가:
  - `OutlinedButton("STT 변환파일 선택")` → 파일 목록/선택기
  - 선택된 파일명 표시
  - `Button("변환 시작")` → 요약방식 선택 BottomSheet

**RecordingViewModel 변경사항**:
- `selectedSttFile: File?` 상태 추가
- `startResummarize(summaryMode: String)` 함수 추가
- STT → 요약만 실행 (STT 단계 건너뜀)
- 파일이름 기본값에 요약방식 라벨 포함

### 2.6 화자이름 변경 기능

**변경 파일**: `MeetingListScreen.kt`, `data/Meeting.kt`, `data/MeetingDao.kt`, `viewmodel/MeetingListViewModel.kt`

#### 2.6.1 개요

STT 변환 결과에서 자동 부여된 `[화자1]`, `[화자2]` 등의 레이블을 실제 참석자 이름으로 변경하는 기능이다. 회의목록 상세 뷰 내에서 직접 변경하며, 변경 내용은 STT 원문 및 요약 텍스트 모두에 일괄 반영된다.

#### 2.6.2 진입 방식

- 회의목록 상세 뷰 상단에 **"화자이름 변경"** 버튼 배치
- STT 탭 또는 요약 탭에서 공통으로 접근 가능

#### 2.6.3 화자이름 변경 다이얼로그

```
┌────────────────────────────────────┐
│         화자이름 변경               │
│────────────────────────────────────│
│                                    │
│  화자1  →  [           ]           │
│  화자2  →  [           ]           │
│  화자3  →  [           ]           │
│                                    │
│────────────────────────────────────│
│      [ 취소 ]      [ 적용 ]        │
└────────────────────────────────────┘
```

**동작 상세**:

1. **화자 자동 감지**: STT 텍스트에서 `[화자N]` 패턴을 정규식으로 추출하여 고유 화자 목록 생성
2. **입력 필드**: 각 화자별 `OutlinedTextField`로 새 이름 입력 (빈칸이면 변경하지 않음)
3. **적용**: 입력된 이름으로 STT 텍스트 및 요약 텍스트 내 `[화자N]` → `[입력이름]` 일괄 치환
4. **저장**: 변경된 텍스트로 로컬 파일 덮어쓰기 + DB의 `sttText`, `summaryText` 필드 업데이트
5. **매핑 기록**: DB `Meeting` 엔티티에 `speakerMap` 필드 추가 (JSON 형식으로 매핑 저장)

#### 2.6.4 DB 스키마 변경

```kotlin
// Meeting.kt
@Entity
data class Meeting(
    // ... 기존 필드 유지
    val speakerMap: String? = null  // JSON: {"화자1":"김대리","화자2":"이과장"}
)
```

- Room 마이그레이션: `ALTER TABLE meetings ADD COLUMN speakerMap TEXT DEFAULT NULL`
- `speakerMap`은 `Gson().toJson(Map<String,String>)` 형태로 직렬화

#### 2.6.5 적용 로직

```
1. STT 텍스트에서 정규식 `\[화자(\d+)\]` 로 고유 화자 추출
2. 다이얼로그에서 이름 입력 → Map<String, String> 생성
   예: {"화자1" → "김대리", "화자2" → "이과장"}
3. STT 텍스트 내 "[화자1]" → "[김대리]" 일괄 치환
4. 요약 텍스트 내 동일 치환 적용
5. 로컬 파일(.txt, .md) 덮어쓰기
6. DB Meeting 레코드 업데이트 (sttText, summaryText, speakerMap)
```

#### 2.6.6 재요약 시 화자이름 유지

- 재요약 실행 시 해당 Meeting의 `speakerMap`이 존재하면, 요약 완료 후 자동으로 매핑 적용
- 사용자가 별도 조작 없이도 이전에 설정한 화자이름이 유지됨

## 5. 기술 고려사항

### 5.1 ChatGPT (OpenAI) API 연동

**Whisper STT**:
- Endpoint: `https://api.openai.com/v1/audio/transcriptions`
- 모델: `whisper-1`
- 최대 파일: 25MB
- 언어 힌트: `language=ko`
- 응답: 일반 텍스트 또는 SRT/VTT 형식
- 화자 분리: 미지원 (Whisper 단독으로는 diarization 불가)

**GPT-4o 요약**:
- Endpoint: `https://api.openai.com/v1/chat/completions`
- 모델: `gpt-4o`
- 기존 프롬프트 템플릿 그대로 사용 (GeminiService 템플릿 공유)

### 5.2 Claude STT 제한사항

현재 Anthropic Messages API는 오디오 파일 직접 입력을 정식 지원하지 않는다. Claude STT 옵션은 UI에 표시하되 **"준비 중"으로 비활성화** 처리하고, API 지원 확인 후 활성화한다.

### 5.3 CLOVA 요약 제한사항

네이버 HyperCLOVA X Studio API를 통한 요약 기능은 별도 계약 및 API 키 발급이 필요하다. 마찬가지로 UI에 표시하되 **"준비 중"으로 비활성화** 처리한다.

### 5.4 SAF (Storage Access Framework) 고려사항

- Android 11(API 30) 이상에서 `MANAGE_EXTERNAL_STORAGE` 없이 사용자 선택 폴더 접근 가능
- `takePersistableUriPermission`으로 앱 재시작 후에도 권한 유지
- `DocumentFile` API로 파일 생성/읽기
- 성능: 대용량 파일 쓰기 시 `ContentResolver.openOutputStream` 사용

### 5.5 STT 엔진별 성능 비교 (SETUP_MANUAL 주의사항 반영 대상)

아래 내용은 V2.0 배포 시 `docs/SETUP_MANUAL.md`에 **"STT 엔진 선택 가이드"** 섹션으로 추가한다.

| 항목 | CLOVA Speech | Gemini | ChatGPT (Whisper) | Claude |
|------|-------------|--------|-------------------|--------|
| 최대 파일 크기 | 200MB | 50MB | 25MB | 미지원 |
| 한국어 정확도 | 최상 (네이버 특화) | 우수 | 양호 | - |
| 화자 분리 | 지원 (최대 8명) | 지원 (프롬프트 기반) | 미지원 | - |
| 처리 속도 | 보통 (1~10분) | 보통 (1~10분) | 빠름 (수 초~수 분) | - |
| API 키 | Invoke URL + Secret Key | Google AI API Key | OpenAI API Key | Anthropic API Key |
| 비용 체계 | 네이버 클라우드 종량제 | 무료 (일일 한도) | 종량제 ($0.006/분) | - |
| 권장 용도 | 긴 회의 (1시간+), 정확도 중시 | 중간 길이, 무료 활용 | 짧은 회의, 빠른 결과 | 향후 지원 예정 |

**주의사항** (매뉴얼 반영):
- CLOVA Speech: 파일 전송 후 서버 처리 시간이 파일 길이에 비례하여 증가. 1시간 녹음 기준 3~5분 소요.
- Gemini: 50MB 초과 파일은 자동 거부. MP3 압축률에 따라 실제 녹음 가능 시간 차이 발생.
- ChatGPT Whisper: 25MB 제한이 가장 작음. 화자 분리가 불가하므로 다수 참석 회의에는 CLOVA 권장.
- Claude: 현재 오디오 직접 입력 API 미지원. UI에 "준비 중" 표시.

### 5.6 하위 호환성

- `formal_text` 모드로 기존 저장된 설정 → 자동으로 `formal_md`로 마이그레이션
- 기존 저장경로의 파일은 이동하지 않음 (사용자 수동 이동 안내)
- DB 스키마 변경: `Meeting` 엔티티에 `speakerMap` 컬럼 추가 (Room 마이그레이션 필요, 기존 데이터 유지)
