## 1. 앱 소개

**회의녹음요약**은 회의 현장에서 녹음 → 음성인식(STT) → AI 요약 → 저장 → 클라우드 업로드까지 원스톱으로 처리하는 Android 앱입니다.

### 핵심 흐름

```
🎙 녹음 시작 (또는 MP3 파일 선택)
      ↓
📝 STT 음성인식 변환 (CLOVA Speech 또는 Gemini)
      ↓
🤖 AI 회의록 요약 (Gemini 또는 Claude, 5가지 방식)
      ↓
💾 파일이름 입력 → 로컬 저장 (.md 파일)
      ↓
☁️ Google Drive 자동 업로드 (설정 시)
```

### 주요 특징

- **듀얼 STT 엔진**: CLOVA Speech(한국어 최적화, 화자분리) + Gemini(멀티모달)
- **듀얼 AI 요약**: Gemini(빠른 처리) + Claude(정교한 분석)
- **5가지 요약 형식**: 상황에 맞는 최적의 요약 방식 선택
- **마크다운 렌더링**: 표, 헤더, 볼드 등 서식이 완벽하게 표시
- **전체화면 보기**: 긴 회의록도 스크롤하며 편하게 확인/복사

## 4. 초기 설정 (API 키)

앱을 사용하려면 **최소 Gemini API 키 1개**가 필요합니다.

### 4.1 Gemini API 키 발급 (필수)

Gemini는 STT 음성인식과 AI 요약 모두에 사용되는 핵심 엔진입니다.

**발급 절차:**

1. 웹 브라우저에서 [Google AI Studio](https://aistudio.google.com/) 접속
2. Google 계정으로 로그인
3. 좌측 메뉴에서 **"Get API Key"** 클릭
4. **"Create API Key"** 버튼 클릭
5. 생성된 키 (AIza로 시작하는 문자열) 복사

**앱에 입력:**

1. 앱 실행 → 하단 탭에서 **⚙ 설정** 선택
2. **"Gemini API Key"** 입력란에 복사한 키 붙여넣기
3. **"저장"** 버튼 클릭
4. "API 키가 저장되었습니다" 메시지 확인

> **참고**: Gemini API는 일일 무료 한도(60회/분)가 제공됩니다. 일반적인 사용에는 충분합니다.

### 4.2 CLOVA Speech API 발급 (권장)

CLOVA Speech는 **한국어 음성인식 정확도가 가장 높은** STT 엔진입니다. 특히 화자분리(누가 말했는지 구분) 기능이 우수합니다.

**발급 절차:**

1. [Naver Cloud Platform](https://console.ncloud.com/) 접속 및 회원가입
2. 콘솔 > **AI·NAVER API** > **CLOVA Speech** 선택
3. **"이용 신청"** 클릭
4. 도메인(앱) 등록:
   - 앱 이름: `회의녹음요약`
   - 서비스 환경: `Android`
5. 생성 완료 후 **Invoke URL**과 **Secret Key** 복사

**앱에 입력:**

1. 앱 > 설정 화면
2. **"CLOVA Invoke URL"** 입력란에 URL 붙여넣기
3. **"CLOVA Secret Key"** 입력란에 키 붙여넣기
4. **"저장"** 클릭

> **권장**: CLOVA Speech를 기본 STT 엔진으로 설정하면 한국어 회의에서 최상의 결과를 얻을 수 있습니다.

### 4.3 Claude API 키 발급 (선택)

Claude는 **더 정교하고 구조화된 요약**이 필요할 때 사용합니다.

**발급 절차:**

1. [Anthropic Console](https://console.anthropic.com/) 접속 및 회원가입
2. **Settings** > **API Keys** > **Create Key**
3. 키 이름 입력 (예: "회의녹음앱") 후 생성
4. 생성된 키 (sk-ant로 시작) 복사

**앱에 입력:**

1. 앱 > 설정 화면
2. **"Claude API Key"** 입력란에 붙여넣기
3. **"저장"** 클릭

### 4.4 설정 완료 체크리스트

| 항목 | 상태 | 비고 |
|------|------|------|
| Gemini API 키 | ☐ | 필수 — STT 및 요약에 사용 |
| CLOVA Speech API | ☐ | 권장 — 한국어 STT 정확도 최상 |
| Claude API 키 | ☐ | 선택 — 고급 요약 시 활용 |
| STT 엔진 선택 | ☐ | CLOVA 또는 Gemini 중 선택 |
| 요약 엔진 선택 | ☐ | Gemini 또는 Claude 중 선택 |
| 요약 방식 선택 | ☐ | 5가지 중 기본값 선택 |

```
┌─────────────────────────────────────────┐
│                                         │
│           (각 탭 콘텐츠)                 │
│                                         │
├─────────────────────────────────────────┤

| 탭 | 기능 |
|-----|------|

## 6. 녹음/변환 사용법

### 6.1 실시간 녹음

**Step 1.** 앱 실행 → 🎙 녹음/변환 탭 선택

**Step 2.** **"녹음 시작"** 버튼 클릭
- 화면 상단에 경과 시간 표시
- 빨간색 표시로 녹음 중임을 확인

**Step 3.** 녹음 중 조작
- **일시정지**: 일시정지 버튼으로 녹음 중단 (재개 가능)
- **중지**: 중지 버튼으로 녹음 종료

**Step 4.** 녹음 중지 후 자동으로 **변환 프로세스** 시작

> **팁**: 마이크를 회의 참석자들 가운데에 놓으면 음성 인식 품질이 향상됩니다.

### 6.2 기존 MP3 파일 선택

이미 녹음된 오디오 파일을 분석할 수 있습니다.

1. **"MP3 선택"** 버튼 클릭
2. 파일 탐색기에서 오디오 파일 선택 (MP3, WAV, M4A, OGG 지원)
3. 선택 후 자동으로 변환 프로세스 시작

> **주의**: CLOVA Speech는 최대 200MB, Gemini는 최대 50MB 파일을 지원합니다. 파일이 클 경우 CLOVA Speech를 사용하세요.

### 6.3 변환 프로세스

녹음 또는 파일 선택 후 다음 순서로 자동 진행됩니다:

```
[1단계] STT 음성인식 변환 중...  ████████░░ 80%
         ↓
[2단계] AI 회의록 요약 중...     ██████░░░░ 60%
         ↓
[3단계] 핵심 지표 추출 중...     ████░░░░░░ 40%
         ↓
[4단계] 파일이름 입력 다이얼로그
         ↓
[완료] 저장 + Drive 업로드
```

### 6.4 파일이름 입력

변환 완료 후 파일이름 입력 다이얼로그가 표시됩니다.

- **기본값**: `2026-0323_회의녹음` 형식으로 자동 생성
- 원하는 이름으로 수정 가능
- 확장자 `.md`는 자동 추가

### 6.5 변환 결과 확인

변환 완료 후 화면에 결과가 표시됩니다:

- **STT 변환 결과**: 음성이 텍스트로 변환된 원문 (미리보기)
- **회의록 요약 결과**: AI가 요약한 회의록 (미리보기)
- **"전체보기"** 버튼: 각 결과를 전체화면으로 확인 가능

## 7. 회의목록 사용법

### 7.1 회의목록 조회

📋 회의목록 탭을 선택하면 저장된 모든 회의록이 날짜순으로 표시됩니다.

각 항목에는 다음 정보가 표시됩니다:
- 회의 제목 (파일이름)
- 저장 날짜/시간
- 요약 방식

### 7.2 상세 보기

회의 항목을 클릭하면 상세 뷰가 표시됩니다.

```
┌─────────────────────────────────────────┐
│  ← 뒤로              2026-03-21 투자심사 │
│─────────────────────────────────────────│
│                                         │
│  ┌─ 회의록 요약 ───────────────────────┐ │
│  │  (요약 내용 미리보기, 최대 6줄)     │ │
│  │                        [전체보기]   │ │
│  └─────────────────────────────────────┘ │
│                                         │
│  ┌─ STT 변환 원문 ─────────────────────┐ │
│  │  (STT 내용 미리보기, 최대 4줄)      │ │
│  │                        [전체보기]   │ │
│  └─────────────────────────────────────┘ │
│                                         │
│          [ 📋 전체 내용 보기 ]          │
│                                         │
└─────────────────────────────────────────┘
```

### 7.3 전체화면 보기

**"전체보기"** 또는 **"전체 내용 보기"** 버튼을 클릭하면:

- 전체화면 다이얼로그가 열립니다
- 마크다운 서식이 적용된 상태로 표시 (표, 헤더, 볼드 등)
- 스크롤하여 전체 내용 확인 가능
- 텍스트를 길게 눌러 선택/복사 가능

### 7.4 회의록 삭제

- 회의 항목을 길게 눌러 삭제 메뉴 표시
- **"삭제"** 선택 후 확인 다이얼로그에서 **"확인"**

> **주의**: 삭제된 회의록은 복구할 수 없습니다. Google Drive에 업로드된 파일은 별도로 삭제해야 합니다.

## 9. 요약 방식 상세 가이드

### 9.1 화자 중심 요약

**추천 상황**: 임원회의, 협상, 팀 토론

각 발화자의 의견과 주장을 개별적으로 정리합니다. "누가 무슨 말을 했는지"가 명확하게 드러납니다.

**출력 예시:**
```
[대표이사]
- 올해 매출 목표 50% 성장 제안
- 신규 제품 개발에 투자 확대 지시

[재무담당]
- 현재 자금 여유가 제한적임을 보고
- 우선순위 검토 필요성 제기
```

### 9.2 주제 중심 요약

**추천 상황**: 정기회의, 프로젝트 리뷰, 부서 회의

논의된 안건(Agenda)별로 내용을 정리합니다. 각 주제에 대한 결론과 실행 과제가 포함됩니다.

**출력 예시:**
```
## Agenda 1: 상반기 매출 목표
- 논의: 50% 성장 목표 제안, 자금 현황 공유
- 결론: 50% 성장 확정, 분기별 중간점검 실시
- 담당: 영업팀

## Agenda 2: 신제품 개발
- 논의: 개발 일정 5월 런칭 검토
- 결론: 5월 15일 런칭 확정
- 담당: 개발팀
```

### 9.3 공식 양식 (마크다운)

**추천 상황**: 경영진 회의, 이사회, 공식 기록 보관

전사 규정을 따르는 정식 회의록 형식입니다. 마크다운 포맷으로 작성되어 표와 서식이 깔끔하게 표시됩니다.

**출력 예시:**
```
# 회의록

| 항목 | 내용 |
|------|------|
| 일 시 | 2026년 03월 21일 14:00 |
| 장 소 | 본사 대회의실 |
| 참석자 | 대표이사, 재무담당, 마케팅담당 |
| 주 제 | 상반기 경영전략 논의 |

## 논의사항
1. 매출 목표 50% 성장안 검토
2. 신제품 개발 일정 확인
...
```

### 9.4 공식 양식 (일반 텍스트)

**추천 상황**: 이메일/메신저 즉시 공유

마크다운 서식 없이 순수 텍스트로 작성됩니다. 어떤 앱에서든 깨짐 없이 표시되므로 즉시 공유에 적합합니다.

### 9.5 강의 요약

**추천 상황**: 교육, 세미나, 일방향 발표

강의/발표 내용의 핵심을 구조적으로 정리합니다. 학습 포인트와 핵심 개념이 명확하게 추출됩니다.

**출력 예시:**
```
## 강의 제목: 디지털 혁신 전략

## 10. Google Drive 연동

### 10.1 최초 설정

1. 설정 화면 > **Google Drive** 섹션
2. **"Google 로그인"** 버튼 클릭
3. Google 계정 선택 및 권한 허용
4. 로그인 완료 후 폴더 지정:
   - **녹음파일 폴더**: MP3 파일 업로드 대상 (기본값: `회의녹음_음성파일`)
   - **회의록 폴더**: 요약/STT 파일 업로드 대상 (기본값: `회의녹음_회의록`)
5. 각 폴더의 **"변경"** 버튼으로 Drive 내 원하는 폴더 선택 가능

### 10.2 자동 업로드

설정이 완료되면 회의록 저장 시 자동으로 Google Drive에 업로드됩니다.

- 녹음 파일(MP3) → 녹음파일 폴더
- 요약 회의록(MD) + STT 원문(MD) → 회의록 폴더

### 10.3 업로드 확인

- 업로드 완료 시 Toast 메시지 표시
- Google Drive 앱에서 해당 폴더 확인 가능

## 13. 문제 해결 (FAQ)

### Q1. STT 변환이 되지 않습니다.

**원인**: API 키가 입력되지 않았거나 만료되었을 수 있습니다.

**해결:**
1. 설정 > API 키 입력 상태 확인
2. 네트워크 연결 확인 (Wi-Fi 또는 모바일 데이터)
3. API 키를 재발급하여 입력

### Q2. 요약 결과가 만족스럽지 않습니다.

**해결:**
1. 다른 요약 방식 선택 후 재시도
2. 다른 AI 엔진으로 변경 (Gemini ↔ Claude)
3. STT 변환 결과가 정확한지 먼저 확인 — STT가 부정확하면 요약도 부정확

### Q3. 마크다운 표가 깨져 보입니다.

**해결:** V1.5에서 WebView 기반 렌더링으로 개선되었습니다. "전체보기" 버튼을 눌러 전체화면에서 확인하면 표가 정상적으로 표시됩니다.

### Q4. Google Drive에 업로드되지 않습니다.

**해결:**
1. 설정 > Google Drive > 로그인 상태 확인
2. 업로드 폴더가 지정되어 있는지 확인
3. Drive 저장공간이 충분한지 확인
4. 네트워크 연결 확인

### Q5. CLOVA Speech가 오래 걸립니다.

**설명:** CLOVA Speech는 서버에서 처리하므로 파일 길이에 비례하여 시간이 소요됩니다. 1시간 녹음 기준 약 3~5분이 소요됩니다. 진행 표시바를 확인하며 기다려 주세요.

### Q6. 녹음 중 앱이 백그라운드로 가면?

**설명:** 녹음 서비스가 Foreground Service로 동작하므로 앱이 백그라운드로 가더라도 녹음은 계속됩니다. 상태바에 녹음 알림이 표시됩니다.

### Q7. 녹음 파일이 너무 큽니다.

**해결:** MP3 형식으로 녹음되므로 1시간 기준 약 30~60MB입니다. Gemini STT는 50MB 제한이 있으므로, 긴 회의의 경우 CLOVA Speech(200MB 제한)를 사용하세요.

### Q8. 화자 분리가 안 됩니다.

**설명:** 화자 분리(누가 말했는지 구분)는 **CLOVA Speech에서만 지원**됩니다 (최대 8명). Gemini STT는 프롬프트 기반으로 화자 구분을 시도하지만 정확도가 낮을 수 있습니다.

## 14. API 비용 안내

### 14.1 Gemini API

- **무료 티어**: 일일 60회 요청 (일반적인 사용에 충분)
- 무료 한도 초과 시 종량제 적용

### 14.2 CLOVA Speech

- **네이버 클라우드 종량제**: 사용량에 따라 과금
- 신규 가입 시 크레딧 제공 (약 10만원 상당)

### 14.3 Claude API

- **종량제**: 입력/출력 토큰 수에 따라 과금
- 일반적인 회의록 요약 1건 기준 약 $0.01~$0.05

> **절약 팁**: STT는 CLOVA Speech(정확도 높음), 요약은 Gemini(무료 한도 활용)로 설정하면 비용을 최소화할 수 있습니다.
