1-3. STEP 2 — Firebase 프로젝트 설정

Google 인증 및 Drive 연동을 위한 Firebase 설정입니다

① Firebase 프로젝트 생성
  1. console.firebase.google.com 접속 → Google 계정 로그인
  2. "프로젝트 추가" 클릭
  3. 프로젝트 이름 입력 (예: meetrecordsummary)
  4. Google 애널리틱스 비활성화 → "프로젝트 만들기"
② Android 앱 등록
  1. 프로젝트 대시보드에서 Android 아이콘 클릭
  2. 패키지 이름 입력: com.krunventures.meetingrecorder
  3. SHA-1 지문 입력 (아래 명령으로 확인)
  4. google-services.json 다운로드
  5. 다운로드한 파일을 프로젝트의 app/ 폴더에 복사
③ SHA-1 지문 확인 방법

PowerShell에서 아래 명령을 실행합니다:

$env:JAVA_HOME = "C:\Program Files\Android\Android Studio\jbr"
& "$env:JAVA_HOME\bin\keytool.exe" -list -v -keystore "$env:USERPROFILE\.android\debug.keystore" -alias androiddebugkey -storepass android

출력에서 SHA1: 뒤의 값을 복사하여 Firebase에 입력합니다.

④ Google Cloud 추가 설정

console.cloud.google.com 접속 후:

  1. Firebase와 동일한 프로젝트 선택
  2. API 및 서비스 → OAuth 동의 화면 → 외부 → 기본 정보 입력 → 저장
  3. API 및 서비스 → 라이브러리 → "Google Drive API" 검색 → 사용 클릭
⚠️ OAuth 동의 화면과 Drive API 활성화를 반드시 완료해야 앱에서 Google Drive 연결이 됩니다.

1-4. STEP 3 — Android Studio 빌드

소스코드를 APK로 빌드합니다

① 프로젝트 열기
  1. Android Studio 실행
  2. File → Open → 다운로드한 프로젝트 폴더 선택
  3. Gradle Sync가 자동으로 시작됩니다 (약 3~5분 소요)
🚫 Gradle Sync 중 AGP 업그레이드 또는 Kotlin 업그레이드 제안이 뜨면 반드시 거부(Don't remind me)하세요. 버전을 변경하면 빌드가 실패합니다.
② google-services.json 확인

app/ 폴더에 google-services.json 파일이 있는지 확인합니다. 없으면 Firebase Console에서 다시 다운로드하세요.

③ 빌드 실행
  1. 상단 메뉴 → Build → Rebuild Project
  2. 하단 Build 탭에서 BUILD SUCCESSFUL 확인
④ APK 파일 추출 (배포용)
  1. 상단 메뉴 → Build → Build Bundle(s)/APK(s) → Build APK(s)
  2. 빌드 완료 후 "locate" 링크 클릭
  3. 생성 경로: app/build/outputs/apk/debug/app-debug.apk
ℹ️ 이 APK 파일을 다른 폰에 전송하면 Android Studio 없이도 직접 설치할 수 있습니다.

1-5. STEP 4 — 폰에 설치

빌드된 앱을 Android 폰에 설치합니다

① 방법 A — Android Studio에서 직접 실행 (권장)
  1. 폰을 USB 케이블로 PC에 연결
  2. 폰에서 "USB 디버깅 허용" 팝업이 뜨면 허용
  3. Android Studio 상단 기기 선택 드롭다운에서 폰 확인
  4. ▶ Run 버튼 클릭 (또는 Shift+F10)
  5. 앱이 자동으로 설치·실행됩니다
② 방법 B — APK 파일 직접 설치
  1. app-debug.apk 파일을 카카오톡, 이메일 등으로 폰에 전송
  2. 폰에서 파일 열기 → "이 소스의 앱 설치 허용" 활성화
  3. 설치 버튼 탭
  4. 설치 완료 후 열기
⚠️ APK 직접 설치 시 "출처를 알 수 없는 앱" 경고가 나타날 수 있습니다. 직접 빌드한 앱이므로 안전하게 설치 허용하시면 됩니다.

1-7. 자주 묻는 질문 (설치편)

Q. Android Studio 없이 APK만 설치하면 안 되나요?
A. APK 파일만 있으면 폰에 바로 설치할 수 있습니다. 단, 최초 1회는 Android Studio에서 빌드하여 APK를 생성해야 합니다. 생성된 APK는 다른 폰에도 전달·설치 가능합니다.
Q. Firebase 프로젝트를 꼭 만들어야 하나요?
A. 네, Google Sign-In (Drive 연동)을 위해 필수입니다. google-services.json 파일이 없으면 앱이 빌드되지 않습니다.
Q. Gradle Sync 중 "AGP 업그레이드" 팝업이 뜹니다
A. 반드시 거부하세요. "Don't remind me again" 또는 "Don't show again"을 선택합니다. 버전을 변경하면 의존성 충돌로 빌드가 실패합니다.
Q. iPhone에서도 사용할 수 있나요?
A. 아니요, 현재 Android 전용입니다. iOS 버전은 계획에 없습니다.
Q. 다른 PC에서 빌드하려면?
A. GitHub에서 소스를 다시 다운로드하고, 동일하게 Firebase 설정(google-services.json) 후 빌드하면 됩니다. SHA-1 지문은 PC마다 다르므로 Firebase에 새 지문을 추가해야 합니다.
Q. Google Drive 연결 시 "개발자 오류 — SHA-1 불일치" 가 나옵니다
A. Firebase Console에 등록한 SHA-1 지문과 현재 PC의 debug.keystore SHA-1이 일치하는지 확인하세요. PC를 변경하거나 Android Studio를 재설치하면 SHA-1이 달라질 수 있습니다.

2-1. API 키 발급 및 입력

STT와 AI 요약에 필요한 API 키를 발급받고 앱에 입력합니다

① CLOVA Speech (STT — 필수)
  1. console.ncloud.com 접속 → 로그인
  2. Services → AI·NAVER API → CLOVA Speech
  3. "이용 신청" → 도메인 생성
  4. Invoke URL과 Secret Key를 복사
  5. 앱 설정 탭 → CLOVA Speech 항목에 붙여넣기
② Gemini API (AI 요약 — 권장)
  1. aistudio.google.com/apikey 접속
  2. "Create API Key" 클릭 → 키 생성
  3. 생성된 키를 복사
  4. 앱 설정 탭 → Gemini API Key 항목에 붙여넣기
③ Claude API (선택)
  1. console.anthropic.com 접속 → API Keys
  2. "Create Key" → 키 복사 → 앱에 입력
④ ChatGPT API (선택)
  1. platform.openai.com 접속 → API Keys
  2. "Create new secret key" → 키 복사 → 앱에 입력
⑤ 엔진 선택

설정 탭 최상단에서 STT 엔진과 요약 엔진을 선택합니다.

2-3. 녹음하기 / 파일 불러오기

회의 음성을 직접 녹음하거나 기존 오디오 파일을 불러옵니다

① 실시간 녹음
  1. 녹음 탭 → 빨간색 녹음 시작 버튼 탭
  2. 파일이름 입력 다이얼로그 → 이름 입력 (선택)
  3. 녹음 진행 중 화면에 경과 시간 표시
  4. 일시정지 → 재개 → 정지 가능
  5. 정지 시 M4A 파일 자동 저장
ℹ️ 녹음 중 전화가 오면 자동으로 일시정지되고, 통화 종료 후 자동 재개됩니다.
② 오디오 파일 불러오기

2-7. 자주 묻는 질문 (사용편)

Q. STT 변환이 너무 오래 걸립니다
A. CLOVA Speech는 서버에서 처리하므로 녹음 길이의 약 1/3~1/2 시간이 소요됩니다. 1시간 녹음 시 약 20~30분 소요됩니다. 인터넷 연결 상태를 확인하세요.
Q. 요약 결과가 만족스럽지 않습니다
A. 다른 요약 방식으로 다시 시도해보세요. 주제 중심 요약은 안건별 정리에, 화자 중심은 소규모 미팅에 적합합니다. 요약 엔진을 Gemini에서 Claude로 변경해볼 수도 있습니다.
Q. 카카오톡으로 공유했는데 파일이 깨져 보입니다
A. v2.1부터 회의록이 PDF로 변환되어 공유되므로 카카오톡에서 바로 깨끗하게 열립니다. 이전 버전을 사용 중이라면 최신 APK로 업데이트하세요.
Q. Google Drive에 자동 업로드가 되지 않습니다
A. 설정 탭에서 Drive 연결 상태가 "✅ 연결됨"인지 확인하세요. 업로드 폴더가 선택되어 있어야 합니다. 인터넷 연결이 불안정하면 업로드가 실패할 수 있습니다.
Q. 녹음 중 앱을 닫으면 녹음이 중단되나요?
A. 아닙니다. Foreground Service로 동작하므로 앱을 닫아도 녹음이 계속됩니다. 상단 알림바에서 녹음 상태를 확인할 수 있습니다.
Q. API 비용이 얼마나 드나요?
A. Gemini API는 무료 한도 내에서 충분히 사용 가능합니다. CLOVA Speech는 NAVER Cloud 종량제로 과금되며, 월 300분 미만이면 소액입니다. Claude와 ChatGPT는 각 회사의 API 요금 정책을 따릅니다.
Q. 데이터가 외부로 유출되지 않나요?
A. 녹음 파일은 폰 로컬에 저장되고, STT/요약 시에만 API 서버로 전송됩니다. Google Drive 업로드는 본인 계정으로만 이루어집니다. 앱 자체에는 별도의 서버가 없습니다.
