# 회의녹음요약 안드로이드 앱 사용자 매뉴얼

---

## 1. 앱 소개

### 1.1 개요

**회의녹음요약**은 Samsung Galaxy S25를 기반으로 설계된 혁신적인 회의 기록 및 분석 솔루션입니다. 본 애플리케이션은 회의의 실시간 녹음, 첨단 음성인식 기술을 통한 텍스트 변환, 그리고 인공지능 기반의 지능형 요약 기능을 제공합니다.

### 1.2 주요 기능

- **실시간 회의 녹음**: 고품질 오디오 캡처로 회의 내용을 완벽하게 기록
- **자동 음성인식 (STT)**: CLOVA Speech 또는 Google Gemini를 통한 정확한 음성 텍스트 변환
- **지능형 요약**: Gemini 또는 Claude AI를 활용한 다양한 형식의 회의록 요약
- **다중 요약 방식**: 화자 중심, 주제 중심, 공식 양식 등 5가지 요약 형식 지원
- **핵심 지표 자동 추출**: 중요 결정사항, 액션 아이템, 주요 논점 자동 추출
- **클라우드 동기화**: Google Drive 연동을 통한 자동 백업 및 공유
- **원터치 위젯**: 홈 화면에서 직접 녹음 시작 가능
- **기존 오디오 지원**: 외부 오디오 파일 임포트 및 분석

## 3. 설치 방법

### 3.1 Android Studio를 통한 빌드 및 설치

#### 3.1.1 사전 준비

1. Android Studio 최신 버전 설치 (https://developer.android.com/studio)
2. Android SDK 레벨 35 이상 설치
3. Samsung Galaxy S25 USB 드라이버 설치

#### 3.1.2 프로젝트 설정

1. 프로젝트 폴더를 Android Studio에서 열기
2. Gradle 파일 동기화 대기 (자동 처리)
3. `Build > Rebuild Project` 실행

#### 3.1.3 빌드 및 설치

1. Samsung Galaxy S25를 USB로 컴퓨터에 연결
2. 기기의 설정 > 개발자 옵션에서 USB 디버깅 활성화
3. Android Studio에서 `Run > Run 'app'` 클릭
4. 기기 선택 후 설치 대기

#### 3.1.4 설치 확인

1. 기기의 홈 화면에서 "회의녹음요약" 앱 아이콘 확인
2. 앱 실행 후 초기 설정 가이드 진행

## 4. 초기 설정 가이드

### 4.1 필수 설정

#### Step 1: Gemini API 키 발급 (필수)

**Gemini**는 회의록 요약 및 음성인식 지원을 위한 필수 API입니다.

1. Google Cloud Console 접속 (https://console.cloud.google.com/)
2. 신규 프로젝트 생성: `프로젝트 선택 > 새 프로젝트`
3. 프로젝트명 입력: "회의녹음요약"
4. API 및 서비스 활성화:
   - "API 및 서비스" > "API 및 서비스 검색"
   - "Gemini API" 검색 및 활성화
   - "Speech-to-Text API" 검색 및 활성화
5. API 키 생성:
   - "인증정보" > "인증정보 만들기" > "API 키"
   - 생성된 키 복사
6. 앱에서 설정:
   - 앱 실행 > 설정 아이콘 > API 설정
   - Gemini API 키 입력
   - 저장 버튼 클릭

**참고**: 처음 Gemini API 사용 시 일일 무료 할당량(60회/분)이 제공됩니다.

#### Step 2: CLOVA Speech API 키 발급 (권장)

**CLOVA Speech**는 한국어 음성인식에 최적화된 STT 엔진으로, Gemini보다 우수한 한국어 인식 성능을 제공합니다.

1. Naver Cloud Console 접속 (https://console.ncloud.com/)
2. 계정 생성 및 로그인
3. 서비스 활성화:
   - 좌측 메뉴 > AI Services > Speech Recognition
   - 활성화 버튼 클릭
4. 인증 정보 생성:
   - Application 새로 생성
   - Client ID, Client Secret 복사
5. 앱에서 설정:
   - 앱 실행 > 설정 아이콘 > STT 설정
   - CLOVA Speech 선택
   - Client ID 및 Client Secret 입력
   - 저장 버튼 클릭

**권장사항**: CLOVA Speech는 한국어 회의 내용 인식에 우수하므로 기본 STT로 설정하기를 권장합니다.

#### Step 3: Claude API 키 발급 (선택)

**Claude**는 고급 요약 알고리즘을 제공하는 선택사항입니다. 더 상세하고 문맥을 고려한 요약이 필요할 경우 활용하십시오.

1. Anthropic Console 접속 (https://console.anthropic.com/)
2. 계정 생성 및 로그인
3. API 키 생성: "Account Settings" > "API Keys" > "Create Key"
4. 생성된 키 복사
5. 앱에서 설정:
   - 앱 실행 > 설정 아이콘 > 요약 설정
   - Claude AI 활성화 (옵션)
   - API 키 입력
   - 저장 버튼 클릭

### 4.2 선택사항 설정

#### Step 4: Google Drive 연동 (선택)

클라우드 동기화를 통해 회의록을 자동으로 백업하고 여러 기기에서 공유할 수 있습니다.

1. 앱 실행 > 설정 아이콘 > 클라우드 연동
2. Google Drive 연동 활성화
3. Google 계정으로 로그인 (앱 권한 요청 시 "허용" 클릭)
4. 동기화 설정:
   - 자동 동기화: 활성화/비활성화 선택
   - 동기화 빈도: 1시간 / 하루 / 주간 중 선택
5. 설정 완료 후 "저장" 클릭

**보안 주의**: Google 계정은 안전한 환경에서만 로그인하십시오. 기기를 분실할 경우 원격으로 계정을 로그아웃할 수 있습니다.

### 4.3 초기 설정 검증

다음 항목들을 확인하여 초기 설정이 완료되었는지 검증하십시오:

- [ ] Gemini API 키 입력 및 테스트 완료
- [ ] CLOVA Speech API 키 입력 및 테스트 완료 (권장)
- [ ] 마이크 권한 활성화 확인
- [ ] 저장소 권한 활성화 확인
- [ ] Google Drive 연동 (선택사항)

## 5. 사용 방법

### 5.1 실시간 회의 녹음

#### 5.1.1 녹음 시작

1. 앱 메인 화면에서 **"녹음 시작"** 버튼 클릭
2. 화면 상단의 빨간색 "REC" 표시 확인 (녹음 중임을 나타냄)
3. 경과 시간과 파일 크기 실시간 표시 확인
4. 음성 레벨 미터를 통해 마이크 입력 레벨 확인

#### 5.1.2 녹음 일시정지

1. 화면 하단의 **"일시정지"** 버튼 클릭
2. 버튼 상태가 "재개"로 변경됨
3. 필요시 **"재개"** 버튼을 클릭하여 계속 녹음

**주의**: 일시정지 중에도 타이머는 계속 진행됩니다.

#### 5.1.3 녹음 중지

1. 화면 하단의 **"중지"** 버튼 클릭
2. 저장 대화상자 표시
3. 파일명 입력 (예: "2026년 3월 경영진 회의")
4. **"저장"** 버튼 클릭
5. 자동으로 다음 화면(STT 변환 화면)으로 진행

**팁**: 중지하기 전에 "회의 주제" 또는 "참석자 정보"를 추가로 입력하면 요약 품질이 향상됩니다.

### 5.2 기존 오디오 파일 불러오기

1. 메인 화면에서 **"파일 임포트"** 또는 **"불러오기"** 버튼 클릭
2. 파일 탐색기에서 분석할 오디오 파일 선택
   - 지원 형식: MP3, WAV, M4A, OGG
3. 파일 선택 후 "확인" 클릭
4. 자동으로 STT 변환 화면으로 진행

**주의**: 매우 큰 파일(>500MB)의 경우 처리 시간이 길어질 수 있습니다.

### 5.3 음성인식 (STT) 변환 시작

#### 5.3.1 STT 변환 진행

1. 녹음 완료 후 자동으로 표시되는 STT 변환 화면에서:
   - STT 엔진 선택 (CLOVA Speech 또는 Gemini)
   - 언어 설정 (기본값: 한국어)
2. **"변환 시작"** 버튼 클릭
3. 진행 바를 통해 변환 진행상황 모니터링

#### 5.3.2 변환 결과 검토

1. 변환 완료 후 텍스트 화면 표시
2. 필요시 다음 기능 활용:
   - **편집 아이콘**: 오류 수정 및 텍스트 편집
   - **복사 아이콘**: 전체 텍스트 복사
   - **공유 아이콘**: 다른 앱으로 공유

#### 5.3.3 변환 재시도

변환 결과가 만족스럽지 못할 경우:

1. **"재변환"** 버튼 클릭
2. 다른 STT 엔진 선택 (예: Gemini로 변경)
3. **"변환 시작"** 클릭

### 5.4 회의록 요약 확인

#### 5.4.1 요약 생성

1. STT 변환 완료 후 자동으로 표시되는 요약 화면에서:
   - 요약 형식 선택 (다음 섹션 참고)
   - 요약 AI 엔진 선택 (Gemini 또는 Claude)
2. **"요약 생성"** 또는 **"분석 시작"** 버튼 클릭
3. 진행 바를 통해 처리상황 모니터링

#### 5.4.2 요약 결과 검토

1. 요약 완료 후 다음 내용 표시:
   - 선택한 형식에 따른 요약본
   - 핵심 지표 (다음 섹션 참고)
   - 스크린샷 및 내보내기 옵션
2. 필요시 텍스트 편집 또는 재생성

#### 5.4.3 요약 재생성

더 나은 요약이 필요할 경우:

1. **"재생성"** 버튼 클릭
2. 다른 요약 형식 또는 AI 엔진 선택
3. **"요약 생성"** 클릭

### 5.5 핵심 지표 확인

#### 5.5.1 자동 추출되는 지표

요약 완료 후 다음과 같은 핵심 지표가 자동으로 추출됩니다:

| 지표 | 설명 |
|------|------|
| **중요 결정사항** | 회의에서 내려진 주요 의사결정 사항 |
| **액션 아이템** | 담당자별 구체적인 실행 과제 |
| **주요 논점** | 회의 중 집중된 주요 주제 |
| **참석자 기여도** | 발언자별 발언 빈도 및 소요 시간 |
| **회의 진행 시간** | 실제 회의 소요 시간 및 분석 시간 |

#### 5.5.2 지표 상세 조회

1. 각 지표 항목 클릭 시 상세 정보 표시
2. 스크린샷 또는 내보내기 가능

## 6. 요약 방식 설명

### 6.1 요약 형식 선택

본 애플리케이션은 다양한 회의 특성에 맞는 5가지 요약 형식을 제공합니다:

#### 6.1.1 화자 중심 요약 (Speaker-Centric Summary)

**목적**: 각 발화자의 주장과 의견을 중심으로 정리

**특징**:
- 참석자별로 의견과 입장을 구분하여 정리
- "○○는 ..., △△는 ..."의 형식으로 표현
- 다양한 관점을 균형있게 제시
- 이견이 있을 경우 명확하게 대비

**활용 사례**:
- 임원진 회의 (의사결정을 위해 다양한 의견 필요)
- 협상 회의 (각 측의 입장 파악 필요)
- 팀 토론 (팀원별 아이디어 수집)

**예시 구조**:
```
[발화자: 대표이사]
- 올해 매출 목표 50% 성장
- 신규 제품 개발에 투자 확대

[발화자: 재무담당]
- 현재 자금 여유가 제한적
- 우선순위 검토 필요

[발화자: 마케팅담당]
- 신제품 런칭 마케팅 준비 완료
```

#### 6.1.2 주제 중심 요약 (Topic-Centric Summary)

**목적**: 회의에서 논의된 주제별로 정리

**특징**:
- 논의 항목 (Agenda Item)별로 구분
- 각 주제별 결론 또는 합의사항 포함
- 시간 순서에 따라 진행
- 실행 과제와 타임라인 명시

**활용 사례**:
- 정기 회의 (정해진 안건 중심)
- 프로젝트 리뷰 (진행사항별 점검)
- 부서 회의 (여러 안건 병렬 처리)

**예시 구조**:
```
## Agenda 1: 2026년 상반기 매출 목표 수립
- 목표: 매출 50% 성장
- 실행 방안: 신규 고객 확보, 기존 고객 관계 강화
- 담당부서: 영업팀
- 기한: 2026년 6월 30일

## Agenda 2: 신제품 개발 일정 확인
...
```

#### 6.1.3 공식 양식 (Markdown 형식)

**목적**: 문서화를 위한 정식 회의록

**특징**:
- 전사 규정을 따르는 공식 회의록 형식
- Header, 참석자, 안건, 결론 등 표준화된 구조
- Markdown 형식으로 다른 문서화 도구와 호환
- 인쇄 또는 아카이빙에 최적화

**활용 사례**:
- 경영진 회의 기록 (정식 문서 보관)
- 이사회 회의 (감사 및 규정 준수)
- 계약 협상 기록 (증거자료 필요)

**예시 구조**:
```markdown
# 회의록

## 회의 정보
- **회의명**: 2026년 상반기 경영진 회의
- **개최 일시**: 2026년 3월 20일 14:00~15:30
- **개최 장소**: 본사 대회의실
- **기록자**: 기획팀장 김순신

## 참석자
- 대표이사 이순신
- 재무담당 이숨
- 마케팅담당 이숨숨

## 안건 및 결론
1. 상반기 매출 목표 수립: 50% 성장 확정
2. 신제품 개발 일정: 5월 런칭 예정
...

## 결정사항
- 매출 목표 50% 성장 지시
- 신제품 개발비 추가 배정

## 액션 아이템
| 항목 | 담당자 | 기한 |
|------|--------|------|
| 신제품 마케팅 계획 수립 | 마케팅팀 | 2026-04-30 |
```

#### 6.1.4 공식 양식 (일반 텍스트)

**목적**: 이메일 또는 메신저로 공유 가능한 일반 텍스트 회의록

**특징**:

## 7. 저장 경로

### 7.1 파일 저장 구조

본 애플리케이션은 다음과 같은 계층적 폴더 구조로 모든 파일을 관리합니다:

```
내장 저장소 (Internal Storage)
├─ Documents/
│  └─ Meeting Recording/
│     ├─ 2026-03-20_경영진회의/
│     │  ├─ 원본_녹음.m4a
│     │  ├─ STT_텍스트.txt
│     │  ├─ 요약_화자중심.md
│     │  ├─ 요약_주제중심.md
│     │  ├─ 요약_공식양식.md
│     │  ├─ 핵심지표.json
│     │  └─ 메타데이터.json
│     ├─ 2026-03-19_팀회의/
│     └─ 2026-03-18_고객미팅/
└─ ...
```

### 7.2 폴더 상세 설명

#### 원본 녹음 파일 (원본_녹음.m4a)
- **위치**: 내장 저장소 > Documents > Meeting Recording > [회의명 폴더]
- **파일명**: `원본_녹음.m4a`
- **설명**: 앱에서 직접 녹음한 원본 오디오 파일 또는 임포트한 오디오 파일
- **용도**: 추후 재검토 또는 외부 분석 필요 시 사용

#### STT 변환 텍스트 (STT_텍스트.txt)
- **위치**: 내장 저장소 > Documents > Meeting Recording > [회의명 폴더]
- **파일명**: `STT_텍스트.txt`
- **설명**: 음성인식으로 변환된 순수 텍스트 (편집 이력 포함)
- **용도**: 녹음 내용의 원본 텍스트 기록

#### 요약 파일들

**화자 중심 요약** (`요약_화자중심.md`)
- 각 발화자별 의견을 구분하여 정리한 Markdown 형식

**주제 중심 요약** (`요약_주제중심.md`)
- 회의 안건별로 구분하여 정리한 Markdown 형식

**공식 양식 요약** (`요약_공식양식.md`)
- 전사 규정에 따른 정식 회의록 Markdown 형식

**일반 텍스트 요약** (`요약_공식양식.txt`)
- 위의 공식 양식을 순수 텍스트로 변환

**강의 요약** (`요약_강의요약.md`)
- 학습 내용 중심의 Markdown 형식

#### 핵심 지표 (핵심지표.json)
- **위치**: 내장 저장소 > Documents > Meeting Recording > [회의명 폴더]
- **파일명**: `핵심지표.json`
- **내용**: 다음 정보를 JSON 형식으로 저장
  - 중요 결정사항 (배열)
  - 액션 아이템 (배열)
  - 주요 논점 (배열)
  - 참석자별 기여도 (객체)
  - 회의 통계 (시간, 발언자 수 등)

#### 메타데이터 (메타데이터.json)
- **위치**: 내장 저장소 > Documents > Meeting Recording > [회의명 폴더]
- **파일명**: `메타데이터.json`
- **포함 정보**:
  ```json

    "회의명": "2026년 상반기 경영진 회의",
    "녹음_일시": "2026-03-20T14:00:00+09:00",
    "회의_시간": "90분",
    "참석자": ["이순신", "이숨", "이숨숨"],
    "녹음_방식": "internal_mic",
    "STT_엔진": "CLOVA_Speech",
    "STT_언어": "korean",
    "요약_엔진": "Gemini",
    "요약_형식": ["화자중심", "주제중심"],
    "생성_일시": "2026-03-20T15:30:00+09:00",
    "Google_Drive_동기화": true

  ```

## 8. 위젯 사용법

### 8.1 원터치 녹음 위젯 추가

#### 8.1.1 위젯 추가 방법 (Samsung One UI 기준)

1. 홈 화면의 빈 공간에 오래 터치 (약 2초)
2. "위젯" 또는 "추가" 옵션 클릭
3. 설치된 앱 목록에서 "회의녹음요약" 찾기
4. 위젯 선택: 다음 중 선택
   - **1x1 위젯**: 원터치 녹음 버튼
   - **2x2 위젯**: 녹음 버튼 + 최근 회의 정보
   - **2x4 위젯**: 풀 기능 위젯
5. 원하는 위치에 배치
6. 홈 화면 밖으로 이동하여 위젯 배치 완료

#### 8.1.2 위젯 크기별 기능

**1x1 위젯 (원터치 녹음)**
- 기능: 한 번의 탭으로 녹음 즉시 시작
- 특징: 가장 컴팩트, 빠른 접근
- 추천: 예측 가능한 시간의 정기 회의

**2x2 위젯 (녹음 + 최근 정보)**
- 기능: 녹음 버튼 + 최근 5개 회의 목록
- 특징: 최근 회의 빠른 접근
- 추천: 자주 사용하는 사용자

**2x4 위젯 (풀 기능)**
- 기능: 녹음 시작, 최근 회의 목록, 저장 공간 정보, 설정 바로가기
- 특징: 모든 주요 기능 한눈에 표시
- 추천: 전문적 사용자, 홈 화면 여유 있을 시

### 8.2 위젯 조작 방법

#### 8.2.1 위젯에서 녹음 시작

1. 홈 화면에서 "회의녹음요약" 위젯 확인
2. 빨간색 "REC" 또는 "녹음 시작" 버튼 탭
3. 화면이 앱으로 전환되며 녹음 시작
4. 위젯에 "녹음 중..." 상태 표시

#### 8.2.2 위젯에서 최근 회의 접근

1. 2x2 또는 2x4 위젯에서 최근 회의 목록 표시
2. 원하는 회의 항목 탭
3. 해당 회의의 상세 화면으로 이동

#### 8.2.3 위젯 설정 변경

1. 위젯에 오래 터치 (약 2초)
2. "위젯 설정" 또는 "설정" 옵션 클릭
3. 다음 항목 커스터마이징 가능:
   - 최근 회의 표시 개수
   - 자동 새로고침 빈도
   - 위젯 테마 (라이트/다크)
4. 설정 저장

### 8.3 위젯 제거 방법

1. 위젯에 오래 터치 (약 2초)
2. "제거" 또는 "삭제" 옵션 클릭
3. 위젯이 홈 화면에서 제거됨

**주의**: 위젯 제거 시 저장된 회의 데이터는 삭제되지 않습니다.

## 9. 트러블슈팅 (FAQ)

### 9.1 마이크 관련 문제

#### Q: 녹음할 때 소리가 잘 안 들려요.

**A: 다음 단계를 확인하십시오:**

1. **마이크 권한 확인**
   - 설정 > 앱 > 회의녹음요약 > 권한
   - "마이크" 권한 활성화 확인
   - 만약 비활성화 상태라면 활성화하고 앱 재시작

2. **기기 설정 확인**
   - 음량 버튼으로 마이크 음량 조정 (화면에 음량 레벨 표시)
   - 마이크 구멍이 먼지로 막혔는지 확인
   - 부드러운 천으로 가볍게 닦기

3. **앱 재시작**
   - 앱을 완전히 종료하고 다시 실행
   - 설정 > 앱 > 회의녹음요약 > "강제 종료"

4. **기기 재부팅**
   - Samsung Galaxy S25 재부팅
   - 전원 버튼 장시간 누르기 > 재부팅

#### Q: 마이크 권한을 어떻게 주나요?

**A: Samsung Galaxy S25에서 마이크 권한 설정:**

1. 설정 앱 실행
2. "앱" 또는 "애플리케이션"
3. "회의녹음요약" 검색 및 클릭
4. "권한" 항목 클릭
5. "마이크" 항목에서 "허용" 선택

**또는 앱 실행 시 권한 요청:**
1. 앱 실행 시 "마이크 권한 필요" 팝업 표시
2. "허용" 버튼 클릭
3. 필요 시 "계속 허용" 선택

#### Q: 여러 명이 있는 회의실에서 모두의 목소리를 잘 녹음하려면?

**A: 다음 권장사항을 따르십시오:**

1. **위치 선정**: 기기를 회의 테이블 중앙에 배치
2. **거리**: 모든 참석자로부터 60~90cm 이내 거리 유지
3. **스피커 사용**: 화상 회의인 경우 스피커 음량을 적당히 조정
4. **배경 소음 감소**: 창문 닫기, 에어컨 음량 낮추기
5. **마이크 방향**: Samsung Galaxy S25의 마이크 위치(보통 기기 하단)가 발화자를 향하도록 배치

### 9.2 API 키 관련 문제

#### Q: Gemini API 키 오류가 발생합니다.

**A: 다음을 확인하십시오:**

1. **API 키 정확성**
   - 복사한 API 키가 완전한지 확인
   - 공백이 추가되지 않았는지 확인
   - 특수문자가 손상되지 않았는지 확인

2. **API 활성화 상태**
   - Google Cloud Console 접속
   - 프로젝트 확인
   - Gemini API 및 Speech-to-Text API 활성화 상태 확인

3. **할당량 초과**
   - Google Cloud Console > Quotas 확인
   - 일일 할당량(60회/분) 초과했는지 확인
   - 필요시 결제 방법 등록 후 할당량 증대

4. **인증 정보 재발급**
   - Google Cloud Console > 인증정보
   - 기존 API 키 삭제
   - 새로운 API 키 생성
   - 앱에서 새 키로 업데이트

#### Q: CLOVA Speech API 키 오류가 발생합니다.

**A: 다음을 확인하십시오:**

1. **Naver Cloud 계정 상태**
   - Naver Cloud Console 로그인 확인
   - 계정 상태 정상 확인
   - 결제 수단 등록 확인

2. **API 활성화 상태**
   - Naver Cloud Console > Services > Speech Recognition
   - 서비스 활성화 상태 확인

3. **인증 정보 정확성**
   - Client ID와 Client Secret 완전성 확인
   - 복사 시 공백 추가 여부 확인

4. **할당량 확인**
   - 일일 API 호출 한도 초과 확인
   - 필요시 Naver Cloud에 할당량 증대 요청

5. **재설정**
   - Naver Cloud Console에서 새로운 Application 생성
   - 새로운 Client ID/Secret 발급
   - 앱에서 새 정보로 업데이트

#### Q: Claude API 키가 작동하지 않습니다.

**A: 다음을 확인하십시오:**

1. **API 키 유효성**
   - Anthropic Console 접속
   - 생성된 API 키 목록 확인
   - 키가 활성화 상태인지 확인 (만료되지 않았는지)

2. **API 호출 한도**
   - Anthropic Console > Usage 확인
   - 월간 한도 초과 확인
   - 필요시 결제 방법 업그레이드

3. **네트워크 연결**
   - Wi-Fi 또는 모바일 데이터 연결 확인
   - 방화벽 또는 VPN이 API 차단하지 않는지 확인

#### Q: 어떤 API 키가 필수인가요?

**A:**

| API | 필수 여부 | 용도 | 대안 |
|-----|---------|------|------|
| Gemini | 필수 | 회의록 요약, 기본 STT | 없음 |
| CLOVA Speech | 권장 | 한국어 STT (고품질) | Gemini STT 사용 가능 |
| Claude | 선택 | 고급 요약 분석 | Gemini 사용 가능 |

최소한 Gemini API 키는 필수입니다.

### 9.3 녹음 관련 문제

#### Q: 녹음이 시작되지 않습니다.

**A: 다음 단계를 실행하십시오:**

1. **저장소 권한 확인**
   - 설정 > 앱 > 회의녹음요약 > 권한
   - "저장소" 또는 "파일" 권한 활성화 확인

2. **저장 공간 확인**
   - 설정 > 저장소 > 가용 공간 확인
   - 최소 1GB 이상 가용 공간 필요
   - 필요시 불필요한 파일 삭제

3. **앱 재시작**
   - 앱 완전 종료 후 재시작

4. **기기 재부팅**
   - Samsung Galaxy S25 재부팅

#### Q: 녹음이 중간에 끊깁니다.

**A: 다음을 확인하십시오:**

1. **저장 공간 부족**
   - 녹음 중 저장소 가용 공간 부족 시 녹음 중단
   - 설정 > 저장소에서 확인
   - 불필요한 파일 삭제 또는 microSD 카드 사용

2. **배터리 부족**
   - 배터리가 15% 이하일 때 기능 제한
   - 배터리 절약 모드 해제
   - 충전 후 다시 시도

3. **백그라운드 앱**
   - 너무 많은 앱 실행 중일 시 성능 저하
   - 다른 앱 종료 후 재시도

4. **네트워크 끊김**
   - Google Drive 자동 동기화 중이면 일시 해제
   - Wi-Fi 연결 안정성 확인

#### Q: 녹음 파일이 저장되지 않았습니다.

**A: 다음을 확인하십시오:**

1. **저장 진행 상황 확인**
   - 녹음 중지 후 "저장" 버튼 클릭 후 진행 바 표시 확인
   - 진행 바가 100% 완료될 때까지 대기

2. **저장 위치 확인**
   - 파일 탐색기 > Documents > Meeting Recording 확인
   - 최근에 생성된 폴더 확인

3. **저장소 권한 다시 확인**
   - 권한 설정 다시 검토

4. **앱 데이터 초기화** (최후의 수단)
   - 설정 > 앱 > 회의녹음요약 > 저장공간 > "데이터 삭제"
   - 앱 재시작 후 초기 설정 다시 진행

### 9.4 STT 관련 문제

#### Q: STT 변환이 시작되지 않습니다.

**A: 다음을 확인하십시오:**

1. **API 키 설정 확인**
   - CLOVA Speech 또는 Gemini API 키가 입력되었는지 확인
   - 설정 > API 설정에서 확인
   - API 키 유효성 테스트 (설정에 테스트 버튼 있음)

2. **네트워크 연결 확인**
   - Wi-Fi 또는 모바일 데이터 연결 확인
   - 신호 강도 확인 (최소 2G 이상)
   - 방화벽 또는 VPN 차단 여부 확인

3. **오디오 파일 확인**
   - 녹음 파일이 정상적으로 생성되었는지 확인
   - 파일 크기가 0 바이트는 아닌지 확인
   - 손상된 파일이 아닌지 확인

4. **STT 엔진 전환**
   - CLOVA Speech 사용 중이면 Gemini로 전환 후 시도
   - 그 반대도 시도

#### Q: STT 결과가 비어있거나 오류 텍스트만 있습니다.

**A: 다음을 확인하십시오:**

1. **오디오 품질 확인**
   - 원본 녹음에 실제 음성이 있는지 확인
   - 음량이 너무 작지 않은지 확인
   - 배경 소음이 심하지 않은지 확인

2. **언어 설정 확인**
   - 설정 > STT 설정 > 언어가 "한국어"로 설정되어 있는지 확인
   - 영어나 다른 언어로 녹음된 음성인 경우 언어 변경

3. **STT 엔진 성능**
   - 다른 STT 엔진 시도 (CLOVA <-> Gemini 전환)
   - 짧은 테스트 녹음으로 기본 동작 확인

4. **파일 포맷 확인**
   - 지원되는 포맷인지 확인 (MP3, WAV, M4A, OGG)
   - 손상된 파일 재확인

5. **재변환 시도**
   - STT 변환 화면에서 "재변환" 버튼 클릭
   - 다른 엔진 선택 후 다시 시도

#### Q: STT 변환이 너무 오래 걸립니다.

**A: 다음을 확인하십시오:**

1. **파일 크기**
   - 매우 큰 파일(>500MB)인 경우 처리 시간 증가
   - 1시간 녹음 기준 보통 5~10분 소요

2. **네트워크 속도**
   - Wi-Fi 속도 확인 (최소 5Mbps 권장)
   - 4G/5G 사용 고려

3. **서버 상태**
   - Naver 또는 Google 서버 상태 페이지 확인
   - 서버 점검 중이 아닌지 확인

4. **기기 성능**
   - 다른 앱 종료하여 리소스 확보
   - 배터리 절약 모드 해제

### 9.5 요약 관련 문제

#### Q: 요약 생성이 실패합니다.

**A: 다음을 확인하십시오:**

1. **STT 결과 확인**
   - STT 변환 결과가 정상적으로 생성되었는지 확인
   - 텍스트가 비어있지 않은지 확인

2. **API 키 설정**
   - Gemini 또는 Claude API 키가 정상 설정되었는지 확인
   - API 할당량 초과 여부 확인

3. **텍스트 길이**
   - 텍스트가 너무 짧으면 요약 불가 (최소 100자 권장)
   - 텍스트가 너무 길면 처리 시간 증가

4. **요약 형식 변경**
   - 다른 요약 형식 선택 후 시도

#### Q: 요약 결과가 마음에 들지 않습니다.

**A: 다음 방법을 시도하십시오:**

1. **다른 AI 엔진 사용**
   - Gemini 대신 Claude 선택 (또는 그 반대)
   - 성능이 다를 수 있습니다.

2. **다른 요약 형식 선택**
   - 5가지 요약 형식 중 다른 형식 시도
   - 회의 특성에 맞는 형식 선택

3. **텍스트 수동 편집**
   - STT 텍스트를 직접 수정 후 재요약
   - 오류나 불분명한 부분 수정

4. **재생성**
   - 요약 화면에서 "재생성" 또는 "재분석" 버튼 클릭
   - 같은 조건으로 재시도하면 때때로 다른 결과 생성

   - 매우 큰 파일(>100MB)인 경우 동기화 시간 증가

## 부록 C. API 비용 추정

### C.1 Gemini API 비용

Gemini API는 월 $2에 가입하면 월 10,000회 요청까지 무료입니다.

초과 시 요금:
- 입력: $0.000075 / 1000자
- 출력: $0.0003 / 1000자

**예시**: 1시간 회의 (약 6,000자) 요약 1회
- 입력: 6,000 × $0.000075 / 1000 = $0.00045
- 출력: 2,000 × $0.0003 / 1000 = $0.0006
- **합계**: 약 $0.0012

### C.2 CLOVA Speech API 비용

Naver Cloud는 월 100,000건까지 무료입니다.

초과 시 요금:
- 인식: 1,000건당 약 ₩500 (~$0.38)

**예시**: 1시간 회의 STT 변환 1회 (약 200분 음성)
- **예상 비용**: 무료 범위 내 (월 100,000건 이내)

### C.3 Claude API 비용

Claude API는 사용량 기준 선불 제도입니다.

가격 (Claude 3.5 Sonnet):
- 입력: $0.003 / 1K tokens
- 출력: $0.015 / 1K tokens

**예시**: 1시간 회의 요약 1회 (약 4,000 tokens)
- 입력: 1,000 tokens × $0.003 = $0.003
- 출력: 3,000 tokens × $0.015 = $0.045
- **합계**: 약 $0.048

### C.4 월간 예상 비용 (1주 5회 회의 기준)

| 구성 | 월 4주 추정 비용 |
|------|-----------|
| **최소 (Gemini만 사용)** | $0 ~ $2 / 월 |
| **권장 (Gemini + CLOVA)** | $0 ~ $3 / 월 |
| **최고 (Gemini + CLOVA + Claude)** | $2 ~ $5 / 월 |

**주의**: 위의 비용은 무료 할당량 초과 시에만 발생합니다.
